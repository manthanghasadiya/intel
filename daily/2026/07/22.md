## 🤖 AI NEWS

### [📄 N-day is Becoming N-Hour. Patching Faster Won't Save You.](https://thehackernews.com/2026/07/n-day-is-becoming-n-hour-patching.html)
**⏱ Read time:** 5 min

Anthropic's red team demonstrated that Claude Mythos Preview can turn 18 Firefox patches into 8 working code-execution exploits autonomously — the first in under an hour after Mozilla shipped the fix. For Windows, it built PoC crashes for 18 of 21 kernel bugs (fastest in 31 minutes) and chained 8 to SYSTEM, at ~$2,000 each. One of those chains was for a bug Microsoft had rated "Exploitation Unlikely," confirming that exploit-difficulty calibrations built for human researchers no longer hold. The window between patch disclosure and weaponized exploit has collapsed from weeks to hours.

**Why it matters:** The patch itself is now the attacker's roadmap. Defenders can no longer rely on patch speed as a primary strategy — validation and controls are now the only lever that works.

## 🔌 MCP / AI AGENTS

### [📄 Open-Source Android AI Agents Could Let Invisible Screen Text Run Code on Host PCs](https://thehackernews.com/2026/07/open-source-android-ai-agents-could-let.html)
**⏱ Read time:** 5 min
**📄 Related paper:** [../../papers/2026/07/android-ai-agent-invisible-text-attacks.md](link)

Researchers demonstrated 7 novel attack chains against 5 open-source mobile agent frameworks (AppAgent, AppAgentX, Mobile-Agent-v3, Open-AutoGLM, MobA). Every framework fell to at least 6 of 7 attacks. Key vectors: (1) invisible 2%-opacity overlay text that all 6 vision models read (GPT-4o, Claude Opus 4.5, Gemini 3 Pro scored 20/20); (2) TOCTOU race between screenshot capture and adb pull (50-500ms window); (3) shell metacharacter injection through adb shell input text (AppAgent's controller uses subprocess.run with shell=True, with insufficient sanitization — calc.exe launched 20/20 trials). The frameworks' code on main branches still contains the vulnerable patterns as of July 17.

**Why it matters:** Mobile agent frameworks are shipping with the same class of injection vulnerabilities as desktop agents — but with the added attack surface of screen overlay attacks, hardware bezel regions, and unauthenticated broadcast channels. Every mobile agent deployment needs the same tool-call sanitization and input validation that Manny's framework requires.

### [📄 Cursor, Codex, Gemini CLI, Antigravity Hit by Sandbox Escapes](https://www.bleepingcomputer.com/news/security/cursor-codex-gemini-cli-antigravity-hit-by-sandbox-escapes/)
**⏱ Read time:** 4 min

Multiple AI coding agent platforms — Cursor, Codex, Gemini CLI, and Antigravity — have been hit by sandbox escape vulnerabilities. The escapes allow attackers to break out of the restricted execution environments these platforms use to run AI-generated code, potentially gaining access to the host system or user data. No CVE details available yet.

**Why it matters:** Sandbox escapes in AI coding agents undermine the entire security model of running untrusted AI-generated code. If the sandbox fails, every code suggestion becomes a potential payload. This directly parallels the agent-tool-use security problems Manny is solving.

### [Insecure Coding Preferences in Long-Term Memory: Security Risks for LLM-based Code Generation](https://arxiv.org/abs/2607.17619)
**Authors:** Yuchen Chen, Wei Cheng, Yuan Xiao, Zhou Yang, Weifeng Sun, Chunrong Fang, Xiang Chen, Baowen Xu, David Lo, Zhenyu Chen | **Published:** July 20, 2026

First systematic study on how insecure coding preferences stored in LLM long-term memory affect code security. Across ChatGPT, Gemini, Qwen, and Grok (5 languages), insecure memories increased vulnerable code generation by 2.7–50.3 percentage points. Insecure memories are hard to overwrite through normal interaction and create a warning-rate lag. Memory-level safety filtering achieves 100% detection and restores baseline behavior.

**Full breakdown:** [papers/2026/07/insecure-coding-preferences-llm-memory.md](../../papers/2026/07/insecure-coding-preferences-llm-memory.md)

## 🔥 HOT TAKE + CONTENT IDEA

Build a MCP server that audits LLM long-term memory for insecure coding preferences.
