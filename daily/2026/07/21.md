## [📄 FakeGit Campaign Uses 7,600 GitHub Repositories to Spread SmartLoader Malware](https://thehackernews.com/2026/07/fakegit-campaign-uses-7600-github.html)
**⏱ Read time:** 5 min | **Category:** MCP Security / Supply Chain Attack
**📄 Related paper:** [FlowGuard: MCP Security Detection](../../papers/2026/07/flowguard-mcp-security-detection.md)

### Simple Version
Researchers at Island discovered nearly 7,600 malicious GitHub repositories — over 800 of them posing as AI skills or MCP servers — that deliver SmartLoader malware (dropping the StealC infostealer). The campaign, named FakeGit, introduces "AgentBaiting": AI agents searching for MCP servers or skills can autonomously discover these malicious repos and follow the attacker's README instructions without any human interaction.

### Technical Deep Dive
The FakeGit operation uses ~6,600 fabricated GitHub profiles to host 7,600 repos, of which 800 impersonate MCP servers and AI skills for Gmail, WhatsApp, Databricks, Jenkins, and Docker. Malicious ZIP archives trigger a LuaJIT loader chain → obfuscated Lua script → SmartLoader → StealC deployment. Island tested Claude Code, Gemini, and ChatGPT and confirmed these agents surface malicious campaign repos unprompted when asked to find free tools. The campaign has recorded 14M+ downloads across GitHub Release assets in ~200 repos. More than 600 campaign listings were also pushed to public registries (LobeHub, Glama, MCP.so, MCP Market).

### Why This Matters
This is the first documented large-scale supply-chain attack specifically targeting the MCP ecosystem at scale, and the first to demonstrate "AgentBaiting" — where AI agents autonomously compromise themselves. Directly relevant to Manny's mcpsec work: MCP server verification, registry sanitization, and sandboxed agent execution are urgent mitigations.

## [📄 Exposed Server Reveals AI-Assisted Phishing Toolkit Behind WebDAV Malware Campaign](https://thehackernews.com/2026/07/exposed-server-reveals-ai-assisted.html)
**⏱ Read time:** 5 min | **Category:** AI-Assisted Cyber Attacks

### Simple Version
Rapid7 found an exposed malware operator's server containing 1,048 files of an LLM-assisted phishing toolkit mid-development. The operator used an open-source AI coding agent (likely Coderrr) to generate lure templates, test filename-spoofing techniques, and document exploit chains targeting CVE-2025-33053 (WebDAV working-directory hijack, CVSS 8.8, in CISA KEV).

### Technical Deep Dive
The toolkit exploited CVE-2025-33053 using `.url` shortcuts pointing at signed Windows binaries (iediagcmd.exe) while redirecting the working directory to an attacker-controlled WebDAV share so `route.exe` loads from the remote share instead of System32. The operator expanded this into 59 `.url` test variants targeting .NET tools (InstallUtil, RegAsm), LOLBAS binaries, and UAC-bypass candidates. The notes included a `_MAPPING.csv`, tiered test ordering, and theory-of-hijack writeups — all exhibiting LLM-generated formatting and emoji-heavy structure. An active campaign against Windows users in Mexico was caught live, using a typosquatted Mexican government ID-lookup site (gobf[.]mx). The delivery panel (Simba Service) sat on the same server with default credentials unchanged.

### Why This Matters
Demonstrates attackers using LLMs to operate "like a modern software product team" — rapid prototyping, comprehensive test suites, and documented exploit development. Shows the democratization of sophisticated tooling via AI coding agents.

## [📄 HollowGraph Malware Hides C2 and Stolen Files in Microsoft 365 Events Dated 2050](https://thehackernews.com/2026/07/hollowgraph-malware-hides-c2-and-stolen.html)
**⏱ Read time:** 4 min | **Category:** Cyber Espionage / Cloud Security

### Simple Version
Group-IB discovered HollowGraph, a .NET DLL espionage implant that uses a hijacked Microsoft 365 calendar as a two-way dead-drop C2 channel. Operator instructions are planted as calendar events dated to 2050-05-13 (buried where no one scrolls), and stolen files are exfiltrated as encrypted event attachments — all through legitimate Microsoft Graph API traffic.

### Technical Deep Dive
The implant has only two commands: `get` and `send`. It never contacts an attacker-owned server — all tasking and exfiltration routes through a compromised mailbox's calendar via Microsoft Graph API. Stolen data is RSA + AES-256 encrypted with separate key pairs for tasking vs. exfiltration. A secondary DNS channel refreshes Entra ID credentials via IPv6 AAAA records from an attacker domain (stored as a `.cfg` file masquerading as a log). HollowGraph is linked to the Cavern backdoor framework attributed to an Iranian MOIS-linked actor (overlapping with MuddyWater/Lyceum/OilRig). Found on 12 machines (3 actively communicating), active June 3–July 9, 2026.

### Why This Matters
Novel C2 technique abusing trusted cloud services — no vulnerability to patch, no attacker infrastructure to block. Highlights the need for application-level Graph API monitoring, calendar event auditing, and client-credential OAuth hygiene. The forensic approach (hunting 2050-dated events) is a creative detection pattern.

### [📄 OpenSSL Silently Fixes 'HollowByte' DoS Vulnerability](https://www.securityweek.com/openssl-silently-fixes-hollowbyte-dos-vulnerability/)
**⏱ Read time:** 3 min | **Category:** Cryptographic Library Security

Okta's red team discovered HollowByte, a DoS vulnerability in OpenSSL where an 11-byte malicious payload triggers buffer pre-allocation up to 131 KB that glibc does not immediately free. By sending waves of connections with randomized sizes, an attacker fragments server memory permanently — only process termination reclaims it. A 1 GB system became unresponsive after 547 MB frozen; a 16 GB system lost 25% of memory. Silently patched in OpenSSL 4.0.1 and backported to 3.6.3/3.5.7/3.4.6/3.0.21.

**Why it matters:** Silent patch pattern (no CVE ID disclosed initially) means unpatched systems are vulnerable without awareness. Affects Apache, NGINX, Node.js, Python, Ruby, PHP, MySQL, PostgreSQL — essentially the entire web infrastructure stack. Manny should verify his agent infrastructure's OpenSSL versions.

### [Adaptive Adversaries: A Multi-Turn, Multi-LLM Benchmark for LLM Agent Security](https://arxiv.org/abs/2607.18063)
**Authors:** Devina Jain, David Hartmann, Chuan Li | **Published:** July 20, 2026

A 21-scenario benchmark for adaptive multi-round attacks against memoryless LLM defenders. Key finding: allowing 15 rounds of adaptive attack raises ASR from 0-1% to 5.4-14.0%. Pooling three frontier attacker LLMs uncovers 1.4-2.2× more unique successful attacks than any single attacker. Claude Opus 4.6 and GPT-5.4 tied in aggregate but differed sharply: Opus hit 60% ASR on one scenario where others stayed at 7%. Released with 945 attack transcripts and 18,422 competition battles.

**Full breakdown:** [papers/2026/07/adaptive-adversaries-multi-turn-multi-llm-benchmark.md](../../papers/2026/07/adaptive-adversaries-multi-turn-multi-llm-benchmark.md)

### [Self-State Attacks on Self-Hosted AI Agents: How Far Can OS Defenses Go?](https://arxiv.org/abs/2607.17986)
**Authors:** Yimeng Chen, Nathanaël Denis, Roberto Di Pietro, Jürgen Schmidhuber | **Published:** July 20, 2026

Defines "self-state attacks" — compromising an AI agent by corrupting its own memory/config files via legitimate OS syscalls. Formalizes a four-axis attack space (Target, Mechanism, Granularity, Temporal) and evaluates OS-level defenses. A layered defense stack (access-control prevention + workload-conditioned detection + periodic backup) covers most cells, but a small residual attack surface remains structurally undetectable at the OS level.

**Full breakdown:** [papers/2026/07/self-state-attacks-self-hosted-ai-agents.md](../../papers/2026/07/self-state-attacks-self-hosted-ai-agents.md)
