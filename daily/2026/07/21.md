### [📄 AWS Kiro Flaw: Hidden Web Page Text Could Rewrite MCP Config and Run Code](https://thehackernews.com/2026/07/aws-kiro-flaw-let-poisoned-web-page.html)
**⏱ Read time:** 5 min
**📄 MCP / Agents cross-reference**

Intezer and Kodem Security discovered that AWS Kiro, Amazon's agentic coding IDE, could be compromised by hidden text on a web page. The attack: one-pixel white text on an API documentation page instructs Kiro to write a malicious MCP server entry into `~/.kiro/settings/mcp.json` using its `fsWrite` tool (no approval required at the time). Kiro auto-reloads `mcp.json`, launches the attacker's server, and the rogue code runs with the developer's privileges. The approval popup was shown but had no effect — the config reloaded regardless. AWS patched in v0.11.130 by marking `mcp.json` as a protected path requiring explicit approval. This is the same class of vulnerability Johann Rehberger demonstrated on Kiro's launch day in July 2025 (writes to mcp.json + .vscode/settings.json), plus a related CVE-2026-10591 for `.vscode/tasks.json` auto-execution.

**Why it matters:** **This is the most directly MCP-relevant story of the day.** MCP config file poisoning via agent tool-use is the exact problem mcpsec is designed to solve. The attack chain — hidden instructions in rendered content → agent writes to MCP config → arbitrary code execution — is the canonical MCP supply-chain attack. The fact that AWS's same-issue pattern recurred across three separate incidents (2025, early 2026, now) confirms that MCP security is a systemic, unsolved problem.

### [📄 OpenSSL Silently Patches 'HollowByte' DoS Vulnerability](https://www.securityweek.com/openssl-silently-fixes-hollowbyte-dos-vulnerability/)
**⏱ Read time:** 3 min

Okta's red team discovered HollowByte — an 11-byte payload triggers buffer pre-allocation up to 131KB that glibc doesn't immediately free. 1GB system became unresponsive after 547MB frozen; 16GB system lost 25% memory. Silently patched in OpenSSL 4.0.1 and backported to 3.x series. Silent patch pattern means many admins won't know to update.

**Why it matters:** Silent patch (no CVE initially) means unpatched agent infrastructure is vulnerable without awareness. Verify OpenSSL versions on all systems running agent tooling.

### [Deterministic Replay for AI Agent Systems](https://arxiv.org/abs/2607.16200)
**Authors:** Rasheed Mudasiru | **Published:** July 21, 2026

Introduces **agrepl**, a Go-based CLI framework for deterministic replay of AI agent executions. Uses a MITM proxy to intercept all external interactions at the transport layer, serializes them as structured traces, and replays in an isolated environment with zero outbound network access. Achieves replay fidelity F=1.0 and median per-step latency reduction of 98.3% across five workloads (n=250). Ships as a single static binary under MIT license.

**Authors:** Sumit Verma, Pritam Prasun, Pritish Kumar | **Published:** July 21, 2026

Closed-loop guardrail pipeline that evaluates LLM agent outputs across eight measurable dimensions and iteratively remediates failures through an evaluate-rewrite-reevaluate loop. Tested on four frontier LLMs and 4,276 content outputs. Goes beyond binary block/allow to active remediation.
