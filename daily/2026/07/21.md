## 🔐 SECURITY

### [📄 SonicWall Zero-Days Exploited to Deliver Custom Malware for Weeks Before Patch](https://www.securityweek.com/sonicwall-zero-days-exploited-to-deliver-custom-malware-for-weeks-before-patch/)
**⏱ Read time:** 4 min

Two SonicWall SMA1000 zero-days (CVE-2026-15409 — critical SSRF; CVE-2026-15410 — high-severity code injection) were exploited in the wild starting June 22, weeks before SonicWall's July 14 advisory. Volexity tracked the attacker as UTA0533 and found they deployed custom malware (KnuckleBall) plus a Java webshell (OrangeTail) and open-source proxy (Suo5). Activity resembles state-sponsored APT behavior. CISA added both to KEV.

**Why it matters:** Demonstrates the window between exploitation start and patch availability (3+ weeks). Edge appliance zero-days with RCE capability are a persistent vector. Relevant for agent-based vulnerability assessment — prioritize edge appliances.

### [📄 OpenSSL Silently Fixes 'HollowByte' DoS Vulnerability](https://www.securityweek.com/openssl-silently-fixes-hollowbyte-dos-vulnerability/)
**⏱ Read time:** 3 min

Okta's red team discovered HollowByte, a DoS vulnerability in OpenSSL where an 11-byte malicious payload triggers buffer pre-allocation up to 131 KB that glibc does not immediately free. By sending waves of connections with randomized sizes, an attacker fragments server memory permanently — only process termination reclaims it. A 1 GB system became unresponsive after 547 MB frozen; a 16 GB system lost 25% of memory. Silently patched in OpenSSL 4.0.1 and backported to 3.6.3/3.5.7/3.4.6/3.0.21.

**Why it matters:** Silent patch pattern (no CVE ID disclosed initially) means unpatched systems are vulnerable without awareness. Affects entire web infrastructure stack. Verify OpenSSL versions on all agent infrastructure.

## 📄 PAPERS

### [Adaptive Adversaries: A Multi-Turn, Multi-LLM Benchmark for LLM Agent Security](https://arxiv.org/abs/2607.18063)
**Authors:** Devina Jain, David Hartmann, Chuan Li | **Published:** July 20, 2026

21-scenario benchmark for adaptive multi-round attacks against memoryless LLM defenders. Allowing 15 rounds raises ASR from 0-1% to 5.4-14.0%. Pooling three frontier attackers finds 1.4-2.2× more unique attacks than any single attacker. Released with 945 attack transcripts and 18,422 competition battles.

**Full breakdown:** [papers/2026/07/adaptive-adversaries-multi-turn-multi-llm-benchmark.md](../../papers/2026/07/adaptive-adversaries-multi-turn-multi-llm-benchmark.md)

### [Self-State Attacks on Self-Hosted AI Agents: How Far Can OS Defenses Go?](https://arxiv.org/abs/2607.17986)
**Authors:** Yimeng Chen, Nathanaël Denis, Roberto Di Pietro, Jürgen Schmidhuber | **Published:** July 20, 2026

Defines "self-state attacks" — compromising an agent by corrupting its own memory/config files via legitimate OS syscalls. Four-axis attack space (Target, Mechanism, Granularity, Temporal). A layered defense stack covers most attacks, but a small residual set is structurally undetectable at the OS level.

**Full breakdown:** [papers/2026/07/self-state-attacks-self-hosted-ai-agents.md](../../papers/2026/07/self-state-attacks-self-hosted-ai-agents.md)
