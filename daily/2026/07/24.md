## 🤖 AI NEWS

### [📄 ChatGPT AgentForger Flaw — OpenAI Fixes Critical Agent CSRF That Deploys Rogue AI Insiders](https://www.securityweek.com/openai-fixes-chatgpt-agent-flaw-that-could-let-attackers-forge-an-ai-insider/)
**⏱ Read time:** 3 min

Zenity disclosed a critical CSRF vulnerability in OpenAI's ChatGPT Workspace Agents (dubbed "AgentForger") that allowed a single phishing link to silently create, authorize, and deploy a persistent autonomous AI agent inside a victim's organization. The attack exploited URL parameter injection in the Builder tool — `initial_assistant_prompt` was auto-submitted without user interaction. The spawned agent could access workspace apps, harvest documents, steal Slack passwords, and send phishing Teams messages on the victim's behalf. OpenAI has fixed the issue server-side.

**Why it matters:** Agent trust failures like AgentForger are the blueprint for how AI supply chain attacks will work. If Manny is deploying agentic systems with auto-provisioning workflows, this is exactly the attack surface to audit.

### [📄 Kimi K3 Agents Found Redis Zero-Days and Built Working RCE Exploit](https://thehackernews.com/2026/07/kimi-k3-agents-found-redis-zero-days.html)
**⏱ Read time:** 2 min

Researchers claimed Kimi K3 autonomous AI agents independently discovered two new zero-day vulnerability classes in Redis (Streams shared-NACK use-after-free and RedisBloom TDigest OOB write) and constructed working RCE exploits across Redis versions 6.2 through 8.8. The agents leveraged RESTORE-based RDB loading to trigger the flaws. Redis has issued patches, though assigned no CVEs. The degree of claimed agent autonomy remains self-reported and unverified.

**Why it matters:** Whether or not the full autonomy claim holds, this is the first high-profile claim of AI agents discovering zero-days and weaponizing them end-to-end. Manny should track this for implications on safety benchmarks and red team automation.

### [📄 NodeBB Patches Eight AI-Found Flaws Exposing Admin Access and Private Chats](https://thehackernews.com/2026/07/nodebb-patches-eight-ai-found-flaws.html)
**⏱ Read time:** 2 min

Aikido's AI security review of the NodeBB forum software uncovered eight vulnerabilities, three exploitable without authentication — including a homepage redirect that exposed the full admin dashboard to any logged-in member, an unauthenticated user-impersonation route to read private messages, and a server-side template injection allowing arbitrary code execution via fediverse integration. All fixed in NodeBB 4.14.2.

**Why it matters:** All eight bugs share the same root cause: authorization checked on the main entry path but not on side routes. This pattern — "authorization surface asymmetry" — is pervasive in AI-audited code and directly relevant to Manny's agent security work.

## 🔐 SECURITY

### [📄 Bing Images SVG Flaws Let Crafted SVGs Run Commands as SYSTEM (CVSS 9.8)](https://thehackernews.com/2026/07/bing-images-flaws-let-crafted-svgs-run.html)
**⏱ Read time:** 3 min

XBOW disclosed two critical Bing Images vulnerabilities (CVE-2026-32194, CVE-2026-32191, both CVSS 9.8) that allowed crafted SVG files to achieve remote code execution as SYSTEM on Microsoft's Bing servers. The attack chain: Bing's reverse image search fetches a user-provided URL → SVG (XML) is passed to ImageMagick → ImageMagick delegates to external programs via shell → shell command injection → SYSTEM-level compromise. Microsoft fixed server-side in March, but XBOW published full mechanics on July 23 after the disclosure hold expired.

**Exploit/PoC:** Craft SVG with embedded ImageMagick MVG/MSL delegate commands targeting the shell parsing layer.

**Why it matters:** The root cause — image-processing pipeline treating untrusted XML as executable input — is exactly the kind of "format confusion" vulnerability that MCP tool-invocation chains are susceptible to. Manny should check whether any MCP server tool chains pass user-controlled content through delegate-style parsers.

## 🔌 MCP / AI AGENTS

### [📄 Claude Cowork Flaw Could Let AI Agent Escape Its VM and Access Mac Files](https://thehackernews.com/2026/07/claude-cowork-flaw-could-let-ai-agent.html)
**⏱ Read time:** 3 min

Researchers disclosed that Anthropic's Claude Cowork (local execution mode) mounts the entire host filesystem (`/`) read-write into the agent's VM at `/mnt/.virtiofs-root`. Combined with the `tc/act_pedit` kernel privilege escalation path, an agent with guest-root access can escape the VM sandbox and access any host data — including SSH keys, cloud credentials, and other sensitive files. Anthropic closed the report as "informative" without a fix for local mode; cloud execution (the new default) avoids the issue.

**Why it matters:** This is the clearest example yet of the fundamental tension in local agent execution: useful agents need host access, but host access is sandbox escape. Manny's agent policies should mandate read-only filesystem mounts and `ProtectSystem=strict` for any locally-executing agent.

### [Toward Cryptographically Verifiable Authorization for Autonomous AI Agents](https://arxiv.org/abs/2607.21325)
**Authors:** M. Llambí-Morillas, D. Fernández-Fernández | **Published:** Jul 23, 2026

Proposes a formal model (CVA) for cryptographically verifiable authorization of AI agent actions using zk-SNARKs. Defines a relation R_CVA binding an agent principal, authorization request, execution context, and policy satisfaction. Introduces key security properties: authorization soundness, principal binding, request binding, policy binding, and replay resistance. Includes a Groth16-based PoC implementation.
