### Problem Statement (Plain English)
The rapid growth of open-source AI infrastructure (model serving engines, agent platforms, MCP ecosystem, and the LLMs themselves) has outpaced security tooling. Most existing tools only handle one layer (jailbreaks OR infrastructure scanning). We need a unified framework that can red-team across all layers of the AI agent stack — infrastructure, protocol/tool, behavior, and model.

### Methodology (Technical)
The paper presents **AI-Infra-Guard**, organized around "layer-paradigm matching" — each layer of the AI agent attack surface gets a detection paradigm suited to its characteristics:
- **Infrastructure layer:** Deterministic rule matching across 75+ AI components with 1,400+ vulnerability rules
- **Protocol/Tool layer:** LLM-driven agentic auditing of MCP servers and agent-skill packages (supply chain auditing)
- **Agent Behavior layer:** Multi-turn black-box agent red teaming
- **Model layer:** Jailbreak harness with 26+ attack operators across 16 datasets

The framework is open-source and designed as a shared foundation for the community. Key innovation: supply-chain auditing of agent skills/MCP servers that extend AI agents.

### Key Results
- Covers 75+ AI components with 1,400+ vulnerability rules
- 26+ jailbreak attack operators across 16 datasets
- First open-source framework spanning all four layers (infrastructure, protocol/tool, behavior, model)
- Includes MCP server and agent-skill supply-chain auditing as a dedicated module

### What's Novel
The "layer-paradigm matching" concept — recognizing that no single detection paradigm works across all attack surfaces — is novel for AI security. This is the only open-source framework to span infrastructure, MCP/tool, behavior, AND model layers. The supply-chain auditing module for MCP servers and agent skills addresses the exact attack vector exploited by the FakeGit campaign.

### What I Learned (Plain English)
AI agent security can't be solved with one tool. You need different approaches for infrastructure configs (rules), MCP servers (agentic auditing), agent behavior (multi-turn interaction testing), and model safety (jailbreak harnesses). A unified framework that covers all four layers is the right architecture, and AI-Infra-Guard proves it's feasible as open source.
