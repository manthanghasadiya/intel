### Problem Statement (Plain English)
Existing security evaluations for agentic AI systems are tied to specific implementations or domains — you can't compare security across different agent architectures (e.g., ReAct vs. Plan-and-Execute vs. custom graph-based). We need a unified evaluation methodology that works across heterogeneous agent systems and adapts dynamically to the system under test.

### Methodology (Technical)
RIFT-Bench uses a **graph representation-driven methodology** with two automated phases:
1. **Discovery Phase** — extracts the system's structure (agent topology, tool inventory, data flows) into a hierarchical representation
2. **Scanning Phase** — deploys adaptive adversarial attacks tailored to the discovered structure, producing a comprehensive evaluation report

The key insight is representing agentic systems as hierarchical graphs, which allows the system to generate structurally-aware adversarial probes that adapt to each system's unique architecture. RIFT-Bench supports evaluation of both the systems themselves and mitigation strategies.

### Key Results
- Evaluated across 45 agentic systems spanning diverse implementations
- Generalizes effectively to heterogeneous agentic architectures
- Supports direct evaluation of mitigation strategies (not just vulnerability detection)
- Dynamic adaptability enables broader coverage than static test suites

### What's Novel
The graph representation-driven approach to red-teaming — instead of a fixed set of probes, RIFT-Bench first reverse-engineers the system's architecture, then crafts probes that target its specific structure. This is fundamentally different from existing benchmarks that use pre-defined test cases. The ability to evaluate mitigations directly (not just find vulnerabilities) is also unique.

### What I Learned (Plain English)
Instead of writing a fixed list of red-teaming prompts, it's smarter to first learn how the agent system is structured, then craft attacks specifically designed for that architecture. A graph-based representation captures the structure well, and the two-phase (discover → attack) pattern is the right approach for evaluating heterogeneous agent systems.
