### Problem Statement (Plain English)
Current MCP security scanners analyze suspicious signals (like credential-like strings in tool metadata) but can't distinguish real vulnerabilities from harmless placeholders. They lack runtime evidence — they see what looks dangerous but don't actually execute it to confirm. We need a system that combines semantic analysis with runtime verification to produce reliable evidence before flagging a risk.

### Methodology (Technical)
FlowGuard implements a five-stage pipeline: (1) **Semantic Risk Triage** — categorizes tool metadata and responses into risk types; (2) **Recon-Guided Payload Narrowing** — reconnaissance narrows the attack surface; (3) **Schema-Valid Probe Generation** — generates probes conforming to MCP tool schemas; (4) **Evidence Adjudication** — executes probes in runtime and evaluates whether actual harmful behavior occurs; (5) **History-Guided Refinement** — iteratively refines probes based on prior results. This bridges the gap between static semantic signals and dynamic execution evidence. The system also performs semantic-only analysis for risks that can't be executed (e.g., metadata-only risks in tool descriptions).

### Key Results
- **Command Injection (execution-based):** F1 score of 0.879
- **File System Access (execution-based):** F1 score of 0.942
- **Latency improvement:** 2.23x faster than existing dynamic scanners
- **Real-world evaluation:** 523 findings across 326 MCP servers
- **Benchmark:** 1,880 MCP cases across 5 vulnerability categories

### What's Novel
First system to combine semantic risk triage with runtime evidence adjudication specifically for MCP. The schema-valid probe generation that respects MCP's tool definition format is novel — previous scanners generated probes that wouldn't actually work against real MCP servers. The history-guided refinement loop iteratively improves detection.

### What I Learned (Plain English)
Static analysis alone is insufficient for MCP security — you need to actually test-execute the suspicious parameters to separate real vulnerabilities from false positives. The gap between "looks dangerous" and "is dangerous" in MCP interactions is wide, and evidence-grounded detection is the right paradigm to bridge it.
